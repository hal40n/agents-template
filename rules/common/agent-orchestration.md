# エージェントオーケストレーション

## 利用可能なエージェント

定義ファイルは `.agents/agents/` に配置（`.claude/agents/` はシンボリックリンク）：

| エージェント | 目的 | 使うタイミング |
| --- | --- | --- |
| `tdd-guide` | テスト駆動開発 | 新機能実装・バグ修正 |
| `code-reviewer` | コードレビュー | コードの作成・変更後 |
| `security-reviewer` | セキュリティ分析 | コミット前・機密コードの変更時 |
| `refactor-cleaner` | デッドコードのクリーンアップ | コードメンテナンス |
| `python-reviewer` | Pythonコードレビュー | Pythonファイルの変更時 |

## エージェントの積極的な活用

ユーザーへの確認なしに即座に使うこと：

1. 複雑な機能リクエスト → 実装前に計画を立ててから進む
2. コードを作成・変更した直後 → **code-reviewer** を使う
3. バグ修正や新機能 → **tdd-guide** を使う
4. セキュリティに関わるコード → **security-reviewer** を使う
5. Pythonファイルの変更 → **python-reviewer** を使う

## 並列タスク実行

独立した作業は必ず並列で実行する：

```markdown
# Good：並列実行
3つのエージェントを同時に起動する：
1. エージェント1：認証モジュールのセキュリティ分析
2. エージェント2：ユーティリティの型チェック
3. エージェント3：既存テストのカバレッジ確認

# Bad：不要なシーケンシャル実行
エージェント1が終わってからエージェント2、その後エージェント3
```

## 複数視点での分析

複雑な問題には、役割を分けたサブエージェントを使う：

- **コードレビュアー** — 正確性・品質の確認
- **セキュリティ専門家** — 脆弱性・リスクの特定
- **一貫性レビュアー** — 既存コードとのスタイル整合性
- **重複チェッカー** — 冗長なコード・ロジックの検出
