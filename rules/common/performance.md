# パフォーマンス最適化

## モデル選択の指針

**Haiku 4.5**（Sonnetの90%の性能、コスト1/3）：

- 高頻度に呼び出す軽量エージェント
- ペアプログラミング・コード生成
- マルチエージェントシステムのワーカーエージェント

**Sonnet 4.6**（最高のコーディングモデル）：

- メインの開発作業
- マルチエージェントワークフローのオーケストレーション
- 複雑なコーディングタスク

**Opus 4.7**（最も深い推論）：

- 複雑なアーキテクチャの意思決定
- 高度な推論が必要なタスク
- リサーチ・分析作業

## コンテキストウィンドウの管理

以下の作業はコンテキストウィンドウの残り20%以内で行わないこと：

- 大規模なリファクタリング
- 複数ファイルにまたがる機能実装
- 複雑なインタラクションのデバッグ

コンテキスト感度が低いタスク（高い使用率でも許容できる）：

- 単一ファイルの編集
- 独立したユーティリティの作成
- ドキュメントの更新
- 単純なバグ修正

## 拡張思考 + プランモード

拡張思考はデフォルトで有効（内部推論に最大31,999トークンを使用）。

拡張思考の制御方法：

- **トグル**：Option+T（macOS）/ Alt+T（Windows/Linux）
- **設定**：`~/.claude/settings.json` の `alwaysThinkingEnabled` で管理
- **予算上限**：`export MAX_THINKING_TOKENS=10000`
- **詳細表示**：Ctrl+O で思考の出力を確認

深い推論が必要な複雑なタスクの場合：

1. 拡張思考が有効であることを確認する（デフォルトでオン）
2. **プランモード**を有効にして構造的なアプローチをとる
3. 複数回の批評ラウンドで徹底的な分析を行う
4. 役割を分けたサブエージェントで多様な視点を得る

## ビルドのトラブルシューティング

ビルドが失敗した場合：

1. エラーメッセージを分析して根本原因を特定する
2. 修正を小さく段階的に行う
3. 修正のたびにビルドを実行して確認する
4. 類似箇所に同じ問題がないかコードベース全体を確認する
