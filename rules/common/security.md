# セキュリティガイドライン

## 必須セキュリティチェック

コミット前に必ず確認すること：

- [ ] シークレットがハードコードされていない（APIキー・パスワード・トークン）
- [ ] すべてのユーザー入力が検証済み
- [ ] SQLインジェクション対策済み（パラメータ化クエリ）
- [ ] XSS対策済み（HTMLのサニタイズ）
- [ ] CSRF保護が有効
- [ ] 認証・認可が確認済み
- [ ] すべてのエンドポイントにレート制限がある
- [ ] エラーメッセージが機密データを漏らしていない

## シークレット管理

- ソースコードにシークレットを絶対にハードコードしない
- 環境変数またはシークレットマネージャーを必ず使う
- 起動時に必須シークレットの存在を検証する
- 露出した可能性のあるシークレットは即座にローテーションする

## セキュリティ問題への対応手順

セキュリティ問題を発見した場合：

1. 即座に作業を止める
2. **`security-reviewer`** エージェントを使う
3. CRITICALな問題を修正してから作業を再開する
4. 露出したシークレットをローテーションする
5. コードベース全体で類似の問題がないか確認する
